Project Architecture

This project is a decoupled Laravel API backend + React SPA frontend hosted together on a single Hostinger shared hosting account. Everything lives inside public_html/ — the Laravel folder is renamed to api/ and the React build files go directly in public_html/.

📁

Phase 3 — Detail
Critical File Contents
Exact content for index.php and .htaccess — can also be created directly on SSH
A
public_html/index.php
Bootstraps Laravel from the api/ folder — custom entry point
<?php use Illuminate\Foundation\Application; use Illuminate\Http\Request; define('LARAVEL_START', microtime(true)); // Laravel is in api/ folder inside public_html $laravel = __DIR__ . '/api'; if (file_exists($maintenance = $laravel . '/storage/framework/maintenance.php')) { require $maintenance; } require $laravel . '/vendor/autoload.php'; $app = require_once $laravel . '/bootstrap/app.php'; $app->handleRequest(Request::capture());
B
public_html/.htaccess
Routes /api/* to Laravel, everything else to React SPA
Options -Indexes Options +FollowSymLinks # Block sensitive files <FilesMatch "\.(env|log|json|lock|sh|sql)$"> Require all denied </FilesMatch> <IfModule mod_rewrite.c> RewriteEngine On RewriteBase / DirectoryIndex index.html index.php # Pass Authorization header to Laravel RewriteCond %{HTTP:Authorization} . RewriteRule .* - [E=HTTP_AUTHORIZATION:%{HTTP:Authorization}] # Handle OPTIONS preflight (CORS) RewriteCond %{REQUEST_METHOD} OPTIONS RewriteRule ^ - [R=204,L] # IMPORTANT: Route /api/* to Laravel BEFORE the real-file check # (api/ folder exists as a real directory, so this MUST come first) RewriteRule ^api/ index.php [QSA,L] RewriteRule ^sanctum/ index.php [QSA,L] # Let real files/dirs pass (assets/, storage/, sw.js, icons, etc.) RewriteCond %{REQUEST_FILENAME} -f [OR] RewriteCond %{REQUEST_FILENAME} -d RewriteRule ^ - [L] # Everything else → React SPA (client-side routing) RewriteRule ^ index.html [L] </IfModule>
⚠ The /api/* routing MUST come before the real-file check. If reversed, Apache sees the api/ directory as a real path and bypasses Laravel entirely — causing all API calls to return 404.

Known Issues & Fixes (From Live Deployment)

🐛
Issue 1 — All /api/* calls return 404
The most common issue — Laravel is not being reached

Cause A: index.php or .htaccess missing from public_html/

# Diagnose ls -la ~/domains/myeoncasuals.com/public_html/index.php ls -la ~/domains/myeoncasuals.com/public_html/.htaccess # Fix: create them using the SSH commands on the previous page

Cause B: .htaccess has wrong rule order — /api/ file check runs before routing

# Diagnose: check the order in .htaccess grep -n "api/" ~/domains/myeoncasuals.com/public_html/.htaccess # The line with "RewriteRule ^api/" must have a LOWER line number than # the line with "RewriteCond %{REQUEST_FILENAME} -f"

Cause C: Wrong path in index.php

head -10 ~/domains/myeoncasuals.com/public_html/index.php # Must show: $laravel = __DIR__ . '/api'; (NOT '/../laravel')
🐛
Issue 2 — Seeder fails: "Data truncated for column estimated_days"

The estimated_days column in shipping_zones is an integer, but the seeder passed a string '5-7'.

# Fix the file directly on server sed -i "s/'estimated_days' => '5-7'/'estimated_days' => 7/" \ ~/domains/myeoncasuals.com/public_html/api/database/seeders/ProductionSeeder.php # Verify the fix grep estimated_days ~/domains/myeoncasuals.com/public_html/api/database/seeders/ProductionSeeder.php # Re-run seeder (safe — uses firstOrCreate, no duplicates) cd ~/domains/myeoncasuals.com/public_html/api php artisan db:seed --class=ProductionSeeder --force
🐛
Issue 3 — Storage symlink fails "No such file or directory"

The Hostinger home directory path is ~/domains/myeoncasuals.com/ — NOT ~/public_html/. Using the wrong path causes the symlink to fail.

# Always use pwd-based variables to get the correct path cd ~/domains/myeoncasuals.com/public_html/api APIPATH=$(pwd) PUBPATH=$(dirname "$APIPATH") ln -s "$APIPATH/storage/app/public" "$PUBPATH/storage" # Verify ls -la "$PUBPATH/storage"
🐛
Issue 4 — chmod fails "No such file or directory"

Using absolute paths like ~/public_html/api/storage fails. Use relative paths from inside the api/ directory instead.

# Navigate first, then use relative paths cd ~/domains/myeoncasuals.com/public_html/api chmod -R 755 storage chmod -R 755 bootstrap/cache
