Chapter 2
Managing Categories
Organize your products with a 3-level category hierarchy
📌 Why categories matter Categories control how products appear in your store navigation. Customers browse by category — so a clear, well-organized hierarchy makes products easier to find. Your site supports 3 levels: Parent → Child → Sub-child.
🌳
Category Hierarchy Explained
Level 1 — Parent
Main top-level category. Appears in the main navigation menu.

Examples: Women, Men, Kids
Level 2 — Child
Sub-category under a parent. Appears as dropdown items in nav.

Examples: Sarees, Kurtas, Lehengas (under Women)
Level 3 — Sub-child
Deepest level. Indented inside dropdown.

Examples: Silk Sarees, Cotton Sarees (under Sarees)
Recommended starting structure for a fashion store:
Women → Sarees → Silk Sarees / Cotton Sarees / Printed Sarees
Women → Kurtas → Anarkali / Straight Cut / Palazzo Set
Women → Lehengas → Bridal / Party Wear
Men → Ethnic → Kurta Pyjama / Sherwani
Kids → Girls / Boys
➕
How to Add a New Category
Go to Admin → Categories → click "Add Category"
Go to Categories
In the left sidebar, click Categories under the CATALOG section.
Click "Add Category"
The button is in the top-right corner of the Categories page. A form will appear at the top of the page.
Fill in Category Details
Complete the form fields below:
FieldWhat to EnterRequired?
Name Category display name (e.g., "Silk Sarees")
The URL slug is auto-generated from the name		 Required
Parent Category Select a parent if this is a child/sub-child category. Leave blank for a top-level (Level 1) category. Optional
Description Short description shown on the category page Optional
Image Square image for category card (JPG/PNG, min 400×400px recommended) Optional
Banner Image Wide banner shown at the top of the category page (1200×300px recommended) Optional
Sort Order Number to control display order. Lower number = shown first. Use 10, 20, 30... to leave room for insertions. Optional
Active Toggle ON to make category visible to customers —
Show in Menu Toggle ON to display in the top navigation menu —
The slug (URL-friendly name) is auto-generated. For "Silk Sarees" it becomes /categories/silk-sarees
Save the Category
Click Save Category. The new category appears in the list immediately.
Add Child Categories (Next Step)
Repeat the process, but this time select the parent category you just created from the "Parent Category" dropdown. This creates a child category under it.
Best practice: Create all Level 1 (parent) categories first, then go back and add Level 2 children, then Level 3 sub-children.
📋
Step-by-Step: Build a Full Category Tree
Example: Adding "Women → Sarees → Silk Sarees"
Create the Parent: "Women"
Name: Women | Parent: None | Show in Menu: ON | Sort Order: 10 → Save
Create the Child: "Sarees"
Name: Sarees | Parent: Women | Show in Menu: ON | Sort Order: 10 → Save
Create the Sub-Child: "Silk Sarees"
Name: Silk Sarees | Parent: Sarees | Show in Menu: ON | Sort Order: 10 → Save
Verify in Navigation
Open your storefront. Hover over Women in the top nav → you'll see Sarees as a dropdown → and └ Silk Sarees indented below it.
If the category doesn't appear, check that both "Active" and "Show in Menu" are toggled ON.

Adding Product Variants (Sizes & Colors)
Use variants when a product comes in multiple sizes, colors, or combinations
Enable Variants
On the Add/Edit Product page, toggle ON "Has Variants". A variant table will appear below.
Click "Add Variant"
A new row appears in the variant table. Fill in each column:
ColumnExampleNotes
SizeS, M, L, XL, Free SizeWhat size/dimension
ColorRoyal BlueColor name (text)
Color Hex#1e3a8aColor swatch (6-digit hex code)
SKUKSS-RB-MUnique code for this variant
Price2999Overrides base price for this variant
Stock25Units available for this variant
Add More Variants
Click "Add Variant" again for each size/color combination. For a saree in 3 colors: add 3 rows, one per color.
If all variants have the same price, only fill the base Sale Price and leave variant Price empty — it will inherit the base price.
Save the Product
Variants are saved along with the product. Customers will see size/color selectors on the product page.
📷
Uploading Product Images
High-quality images are key to conversions
Save the Product First
Images are uploaded after the product is created. Save basic info first, then scroll down to the Images section.
Upload Images
Click the upload area or drag-and-drop JPG/PNG files. You can upload multiple images at once.
Recommended size: 800×1000px (portrait, 4:5 ratio works best). Maximum file size: 2MB per image.
Set the Primary Image
The first image becomes the primary product thumbnail shown in listings. Drag to reorder images if needed.
Add 4–6 images: front view, back view, close-up of fabric/design, and a styled lifestyle shot.
📊
Bulk Import Products via CSV
Add hundreds of products at once using a spreadsheet
Download the Template
Go to Admin → Products and click "Import", then download the CSV template. This gives you the correct column format.
Fill the Spreadsheet
Open in Excel or Google Sheets. Fill one product per row. Required columns: name, category_slug, base_price, sale_price, status.
The category_slug column must match an existing category slug exactly (e.g., "silk-sarees"). Create categories first before importing.
Upload and Import
Save as CSV, then upload the file via the Import button. Products are created immediately.
Chapter 4
Inventory Management
Monitor and update stock levels for all product variants
📋
Updating Stock Levels
Admin → Inventory
Go to Inventory
Click Inventory in the left sidebar. You'll see all product variants in a table.
Search for a Product
Use the search box to find a product by name. All its variants will be listed.
Edit Stock
Click the Edit icon next to a variant. Type the new stock number and click Save.
Watch for Alerts
The dashboard shows a Low Stock Items count when any variant drops below 5 units. Restock before it hits zero.
When stock = 0, customers see "Out of Stock" and cannot add to cart. Keep buffer stock of at least 5 units per variant.
🟡 Low Stock Warning
Highlighted in yellow when stock < 5 units. Reorder immediately.
🔴 Out of Stock
Highlighted in red with "Out" badge. Product hidden from "Add to Cart" on storefront.

📦
Step-by-Step Order Processing
Admin → Orders
Check for New Orders
Go to Admin → Orders. You'll see a pipeline view. The "New Orders" column shows all pending orders waiting for your action.
Check orders at least twice a day — morning and afternoon. Enable browser notifications if available.
Review the Order
Click View on any pending order to see: customer details, items ordered, delivery address, payment method (COD/Online), and payment status.
If payment method is "Online" and payment status is "Pending", the customer hasn't paid yet. Wait for payment before accepting.
Accept the Order
Click "Accept" on the order card. Status changes from Pending → Confirmed. An SMS/WhatsApp notification link is shown for you to notify the customer.
Pack the Order
Physically pack the items. Then click "Mark as Packed" in the order. Status becomes Packed.
At this stage you can download a Manifest CSV listing all "Ready to Ship" orders — useful for handing to courier partner.
Add Tracking & Dispatch
Once the courier picks up the parcel, click "Dispatch". A form appears to enter:
FieldExample
Courier NameDelhivery / BlueDart / DTDC / Ekart
Tracking NumberDEL1234567890
Tracking URLhttps://www.delhivery.com/track/...
Get the tracking number from your courier partner's app/dashboard. Customers can track their order using this number.
Order Delivered
When the courier delivers the parcel, manually update the status to Delivered from the order detail page. For COD orders, collect payment is confirmed at this step.
📱 WhatsApp Quick Notification Each order card has a WhatsApp link pre-filled with the customer's name and order number. Click it to open WhatsApp Web and send a quick confirmation message.
Chapter 6
Managing Customers
View customer profiles, purchase history, and account management
👥
Customer Management
Admin → Customers
Search for a Customer
Use the search bar to find by name, email, or phone number. Filter by All / Active / Blocked tabs.
View Customer Details
Click a customer row to see: full profile, all past orders, saved addresses, and spending summary (total orders, total spent, average order value).
Block / Unblock a Customer
If a customer is fraudulent or abusive, click Block. They can no longer log in. You can unblock anytime.
Blocking is permanent until you manually unblock. Blocked customers see a "Your account has been suspended" message.

Chapter 8
Banners & Homepage Content
Control what customers see on your homepage hero slider and promotional sections
🖼️
Banner Positions Explained
🦸 Hero
Full-width slider at the top of the homepage. This is the first thing customers see. Multiple hero banners auto-rotate. Recommended size: 1440×600px
📑 Category
Promotional banner within the category listing page. Used for seasonal offers on specific categories. Recommended: 1200×300px
💬 Popup
Modal popup shown to customers (e.g., newsletter signup, sale announcement). Use sparingly — no more than 1 active popup.
📰 Strip
Thin horizontal banner (like the top bar). Good for shipping offers or discount codes. Recommended: 1440×80px
➕
Adding a Hero Banner — Step by Step
Admin → Banners → "+ Add Banner"
Click "+ Add Banner"
In Admin → Banners, click the button at the top right. A form appears.
Fill in Banner Details
FieldWhat to Enter
TitleMain heading on the banner. E.g., "New Festive Collection"
SubtitleSupporting text. E.g., "Discover Premium Ethnic Wear"
Link URLWhere the button takes the customer. E.g., /categories/sarees

Chapter 9
Coupons & Discounts
Create promotional codes to drive sales and reward customers
🏷️
Types of Coupons
% Percentage
Takes a % off the cart total.
Example: SAVE20 → 20% off any order
Set Max Discount to cap the savings (e.g., max ₹500 off)
₹ Flat Amount
Takes a fixed amount off.
Example: FLAT200 → ₹200 off orders above ₹999
🚚 Free Shipping
Removes the shipping charge.
Example: FREESHIP → No delivery charges
🎁 BOGO
Buy One Get One free.
Example: BOGO2024 → Buy any kurta, get one free
➕
Creating a Coupon — Step by Step
Admin → Coupons → "+ Add Coupon"
Enter Coupon Code
The code customers will type at checkout. Use uppercase letters, no spaces. E.g., WELCOME10, DIWALI25, FLAT500
Set Type and Value
Select Percentage, enter Value: 10 (means 10% off). Or select Flat, enter Value: 200 (means ₹200 off).
Set Restrictions
FieldWhat it does
Min Order ValueCoupon only works if cart total ≥ this amount. E.g., ₹999 means coupon needs min ₹999 cart.
Max Discount(For % coupons) Maximum discount given. E.g., 20% off but max ₹500.
Usage LimitTotal uses allowed across all customers. E.g., 100 = first 100 customers can use it.
First Order OnlyToggle ON to restrict to customers who haven't ordered before.
Expires AtAuto-deactivates after this date. Leave blank for no expiry.
Save and Share
Click Save. Share the code on WhatsApp, Instagram, or email campaigns. Customers type it in the cart page.
💡 Welcome Coupon Best Practice Create a coupon code WELCOME10 (10% off, first order only, no expiry). Display it prominently in the site's top bar strip. This converts first-time visitors into buyers.
Chapter 10
Customer Reviews
Moderate and manage product reviews from customers
⭐
Review Moderation
Admin → Reviews
How it works: When a customer submits a review on a product page, it goes to Pending status and is NOT shown on the storefront until you approve it.
Go to Reviews → Pending Tab
Filter to see only pending reviews. Each row shows: product name, customer, star rating, review title and body, and date submitted.
Read the Review
Check if the review is genuine, relevant to the product, and doesn't contain inappropriate content.
Approve even mixed reviews (3-4 stars) — a mix of positive and critical reviews builds customer trust better than only 5-star reviews.
Approve or Reject
Click Approve to publish it on the product page, or Reject if it's spam/inappropriate.

Chapter 13
Reports & Analytics
Understand your business performance with 6 built-in report types
📈
Available Reports
Admin → Reports — all reports have date range filters and CSV export
📊 Sales Report
Total revenue, orders placed, average order value, discounts given. Bar chart of daily revenue. Use for monthly performance reviews.
💰 Profit Report
Revenue minus cost price = gross profit. Profit margin %. Requires cost_price on products. Shows most profitable products.
🧾 GST Report
Calculate GST collected by rate (5%, 12%, 18%, 28%). Shows CGST and SGST breakdown. Export for CA/accounting use.
📦 Products Report
Units sold per product, revenue contribution, average rating. Identify your best-sellers and slow movers.
👥 Customers Report
Orders per customer, total lifetime spend. Identify your top repeat customers for loyalty rewards.
📋 Inventory Report
Out of stock items, low stock alerts. Use before major sale events to ensure enough stock is available.
📅 Monthly Routine: Run the Sales Report for the previous month on the 1st. Export GST Report quarterly for your accountant. Check Inventory Report before every sale event.

🔍
Pincode Checker
Customers verify delivery availability before ordering

On every product page, there's a pincode checker. The customer enters their pincode and sees:

  • Whether delivery is available to their area
  • Estimated delivery time (e.g., "5-7 business days")
  • Shipping charge for their zone
This is powered by your Shipping Zones configuration (Admin → Shipping). Make sure at least one shipping zone is active and set to cover all pincodes.
