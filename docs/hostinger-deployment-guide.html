How Files Are Organized on Hostinger

Hostinger gives you a home directory. Only public_html/ is accessible from the browser. The Laravel app lives outside it for security.

# Hostinger Server Structure /home/u123456789/ # Your Hostinger home folder ├── laravel/ # ← Laravel app (PRIVATE - not browser-accessible) │ ├── app/ │ ├── bootstrap/ │ ├── config/ │ ├── database/ │ ├── routes/ │ ├── storage/ │ │ └── app/public/ # uploaded images live here │ ├── vendor/ │ ├── .env # ← your production environment config │ └── artisan │ └── public_html/ # ← myeoncasuals.com web root (PUBLIC) ├── index.php # ← from deploy/index.php ├── index.html # ← React SPA (from laravel/public/) ├── assets/ # ← Vite-built JS & CSS ├── .htaccess # ← from deploy/public_html.htaccess ├── storage/ # ← SYMLINK → ../laravel/storage/app/public ├── sw.js # service worker (PWA) ├── manifest.webmanifest └── favicon files...
⚠️ Why this structure? Putting Laravel files inside public_html/ exposes your .env (database passwords, API keys) to the internet. Keeping Laravel one level up protects all sensitive files.
Phase 1
Hostinger Account Setup
Configure PHP version, create database, and get SSH access
🐘
Step 1 — Set PHP Version to 8.2
Laravel 12 requires PHP 8.2 or higher
Log in to Hostinger hPanel
Go to hpanel.hostinger.com → log in with your account
Open Hosting → Manage
Click on your hosting plan → click Manage button next to myeoncasuals.com
Find PHP Configuration
In hPanel, go to Advanced → PHP Configuration (or search "PHP" in the search bar)
Select PHP 8.2 or 8.3
From the dropdown, choose PHP 8.2 or 8.3. Click Save.
✓ Also verify these PHP extensions are enabled: BCMath, Ctype, Fileinfo, JSON, Mbstring, OpenSSL, PDO, PDO_MySQL, Tokenizer, XML, GD, ZIP
🗄️
Step 2 — Create MySQL Database
hPanel → Databases → MySQL Databases
Go to Databases Section
In hPanel, find Databases → MySQL Databases
Create a New Database
Enter a database name (e.g., Myeon Casuals) → click Create. Hostinger will prefix it with your account username automatically.
📋 Write down the full database name shown (e.g., u123456789_Myeon Casuals)
Create a Database User
In the same section, create a new user (e.g., Myeon Casuals_user) with a strong password.
📋 Write down: username (with prefix) and password
Assign User to Database
In "Add User to Database" section, select your user and database → grant All Privileges → Save
📋 Note these down — needed for .env file: DB_DATABASE = u123456789_Myeon Casuals  |  DB_USERNAME = u123456789_Myeon Casuals_user  |  DB_PASSWORD = (your password)
🔑
Step 3 — Enable SSH Access
Required to run artisan commands and create storage symlink
Go to Advanced → SSH Access
In hPanel, search for SSH Access. Toggle it ON.
Note Your SSH Credentials
hPanel will show you:
  • Host: (your server IP or hostname)
  • Port: 22 (or 65002 for some plans)
  • Username: your hPanel username (u123456789)
  • Password: your hPanel password
Test SSH Connection
Open Command Prompt / Terminal on your PC and test:
ssh u123456789@your-server-hostname # replace with your credentials
On Windows: Use PowerShell, Windows Terminal, or download PuTTY from putty.org
📧
Step 4 — Create Email Account (for order notifications)
hPanel → Emails → Email Accounts
Create noreply@myeoncasuals.com
Go to Emails → Email Accounts → Create account noreply@myeoncasuals.com with a password.
Note SMTP Settings
Hostinger email SMTP settings (used in .env):
MAIL_HOST=smtp.hostinger.com MAIL_PORT=587 MAIL_USERNAME=noreply@myeoncasuals.com MAIL_PASSWORD=your-email-password

📝
Prepare the Production .env File
Fill in your Hostinger database and email credentials
Open deploy/env.production
File location: C:\laragon\www\last-ecom\deploy\env.production
Open it in Notepad or VS Code.
Fill in ALL the FILL fields

Phase 3
Upload Files to Hostinger
Upload via File Manager or FTP — two separate upload operations
📦 Two Uploads Required:
  1. Upload Laravel app files → into ~/laravel/ (above public_html)
  2. Upload Public files (React build + entry point) → into ~/public_html/
📂
Upload 1 — Laravel App Files
Goes to ~/laravel/ — NOT inside public_html
Go to Hostinger File Manager
hPanel → Files → File Manager. You'll see your home directory with public_html/ folder.
Create a "laravel" folder
In the home directory (NOT inside public_html), create a new folder named laravel
📌 Final path: /home/u123456789/laravel/ — at the same level as public_html, not inside it
Zip the Laravel project on your PC
On your PC, zip the entire laravel/ folder EXCLUDING laravel/public/, laravel/node_modules/ (if any).
# In PowerShell (excludes public/ folder): cd C:\laragon\www\last-ecom Compress-Archive -Path laravel\* -DestinationPath laravel_deploy.zip ` -CompressionLevel Optimal # Note: upload the zip, then delete laravel/public from server
💡 Alternatively, use FTP software like FileZilla — it's easier for large uploads. See FTP section below.
Upload and Extract in File Manager
In File Manager, open the laravel/ folder you created. Upload the zip → right-click → Extract.
Upload the .env file
Upload your filled deploy/env.production to ~/laravel/ and rename it to .env (with the dot, no extension).
⚠️ The .env file must be inside laravel/ — NOT in public_html. It contains your database passwords.
📋 What to upload to ~/laravel/ — checklist:
  • app/ folder
  • bootstrap/ folder
  • config/ folder
  • database/ folder
  • resources/ folder
  • routes/ folder
  • storage/ folder (with empty .gitignore files preserved)
  • vendor/ folder (large — ~50MB, all Composer packages)
  • artisan file
  • composer.json and composer.lock
  • .env (renamed from deploy/env.production)
❌ Do NOT upload: laravel/public/ — this goes to public_html instead
🌐
Upload 2 — Public Files (React Build)
Goes to ~/public_html/ — this is what browsers access
Open public_html in File Manager
Navigate to public_html/ folder. You'll see the default Hostinger placeholder page — delete it.
Upload contents of laravel/public/
From your PC, upload everything inside C:\laragon\www\last-ecom\laravel\public\ — but replace the two files below:
⚠️ Replace these 2 files with the deploy/ versions:
  • Upload deploy/index.php → rename to index.php in public_html (replaces the Laravel one)
  • Upload deploy/public_html.htaccess → rename to .htaccess in public_html
📋 What should be in ~/public_html/ after upload:
  • index.php (from deploy/index.php — NOT the Laravel version)
  • index.html (React SPA — from laravel/public/index.html)
  • .htaccess (from deploy/public_html.htaccess)
  • assets/ folder (Vite-built JS + CSS)
  • sw.js (PWA service worker)
  • manifest.webmanifest
  • registerSW.js
  • favicon files (pwa-64x64.png, apple-touch-icon-180x180.png, etc.)

Using FTP Instead (Recommended for large uploads)

Phase 5
SSH Configuration & Final Setup
Run the setup script to configure permissions, symlink, and caches
🔧
Run All Setup Commands via SSH
One-time setup after uploading files

Connect to SSH first:

ssh u123456789@your-server-hostname

Option A — Run the automated setup script

# Upload deploy/hostinger-setup.sh to ~/ then run: bash ~/hostinger-setup.sh

Option B — Run commands one by one

1. Set Storage Permissions

chmod -R 755 ~/laravel/storage chmod -R 755 ~/laravel/bootstrap/cache

2. Create Storage Symlink (so uploaded images are accessible from browser)

# Remove old symlink if exists rm -f ~/public_html/storage # Create new symlink ln -s ~/laravel/storage/app/public ~/public_html/storage # Verify it worked ls -la ~/public_html/storage # Should show: storage -> /home/u123456789/laravel/storage/app/public

3. Build Laravel Caches (makes the site faster)

cd ~/laravel php artisan config:cache # caches all config files php artisan route:cache # caches route definitions php artisan view:cache # caches blade views

4. Verify Everything

# Check that .env is correct php artisan about # Test database connection php artisan db:show # Check storage symlink ls -la ~/public_html/ | grep storage

Enable SSL (HTTPS) — Free on Hostinger
Required for online payments and PWA install
Go to SSL in hPanel
hPanel → Security → SSL Certificates → find myeoncasuals.com
Install Free SSL (Let's Encrypt)
Click Install next to the free SSL option. Hostinger auto-renews it every 90 days.
Force HTTPS Redirect
In hPanel → Security → SSL → enable "Force HTTPS" toggle. This redirects all http:// traffic to https://
⚠️ Razorpay payments ONLY work on HTTPS. Make sure SSL is active before testing payments.
🌍

Get Hostinger Nameservers
In hPanel → Domains → your domain → you'll see the nameservers:
ns1.dns-parking.com and ns2.dns-parking.com (or Hostinger's nameservers)

Common Errors & Fixes
ProblemLikely CauseFix
500 Internal Server Error Wrong .env, missing APP_KEY, or permissions Check laravel/storage/logs/laravel.log for details. Verify APP_KEY is set. Run chmod 755 storage/
404 on page refresh .htaccess not working Make sure deploy/public_html.htaccess was uploaded as .htaccess. Check mod_rewrite is enabled in PHP config.
API calls fail (CORS error) Wrong APP_URL in .env or SANCTUM config Set APP_URL=https://myeoncasuals.com in .env. Run php artisan config:cache after changes.
Images not showing Storage symlink missing SSH: run ln -s ~/laravel/storage/app/public ~/public_html/storage
Database connection failed Wrong DB credentials in .env Double-check DB_DATABASE, DB_USERNAME, DB_PASSWORD from Hostinger. DB_HOST must be localhost
White screen / blank page PHP version mismatch or missing extension Set PHP to 8.2 in hPanel. Enable PDO_MySQL, Mbstring, OpenSSL extensions.
Razorpay payment fails HTTP (not HTTPS) or wrong keys Enable SSL and use rzp_live_ keys (not rzp_test_)
Admin login fails Session cookie domain mismatch Set SESSION_DOMAIN=myeoncasuals.com in .env. Clear config cache.
